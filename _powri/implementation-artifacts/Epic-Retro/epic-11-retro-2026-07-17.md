## What Went Well

1. **Content-first pipeline (11.1) unblocked everything cleanly.** Geo re-verification caught coordinate drift in 16/20 resorts before any map code shipped. Curated schema with soft vs hard validation split (`Zod` fail on category, warn on highlight length) matched PO editorial workflow.
2. **`web/src/lib/places/` module with exhaustive unit tests.** Category map, field mask, normalizer, TTL, cache, dedupe — each file paired with tests. Downstream stories (11.7 chip changes, 11.8 radius) touched small surfaces without regressions.
3. **Lazy-load pattern reused from `GettingThereSection`.** IntersectionObserver + `next/dynamic({ ssr: false })` kept Mapbox/Leaflet out of initial detail bundle (NFR-16).
4. **`categoryVisuals.ts` as single source of truth (11.6).** Pin colors and chip selected-state colors stay in sync — pattern extended in 11.7 for `ski_resort`.
5. **Follow-up story pattern (amend-not-reopen).** 11.4 tracked QA bugs separately from 11.3 sign-off; 11.8 superseded 11.2 radius without rewriting 11.2's historical record — preserves audit trail.
6. **Mapbox migration (11.5) executed with architecture doc sync.** CSP gotchas (`worker-src blob:`) documented in story dev notes; PO manual browser QA caught silent CSP failures build wouldn't show.
7. **Analytics contract stable across chip model change.** `place_source` + chip-key `place_category` updated through 11.7; `test:analytics` stayed green.
8. **`PoiDetailContent` extraction (11.9).** Shared detail markup between mobile sheet and desktop rail prevents drift — direct response to 11.4's unified display fields.

## Challenges & Growth Areas

1. **Massive scope expansion via PO follow-ups mid-epic.** Original shard planned 3 stories; shipped 9. Each follow-up was valuable product polish (Mapbox, pin shapes, chip UX, desktop rail) but epic planning underestimated PO iteration cycles on a visual, map-heavy feature.
2. **Manual QA caught a ship-stopper Google Maps URL bug (11.4).** `googleMapsPlaceUrl(placeId)` without `query` redirected curated myoko restaurant to Bangkok. Unit tests existed for URL helper but didn't assert both params together until after QA failure.
3. **Curated schema evolved post-11.3 merge.** `place_name` became required in 11.4 — content authors need clear migration path as more resorts get curated entries beyond myoko pilot.
4. **Cross-breakpoint layout regression (11.9).** Mobile grid fix (`grid-cols-1` removal) broke desktop rail animation until CSS cascade ownership was clarified. Lesson documented in 11.9 story file — needs process adoption.
5. **UX spine docs lagged implementation.** `design.md` still referenced rating in POI sheet; architecture said Leaflet until 11.5 updated it; desktop rail has no formal UX spec — dev stories noted gaps but didn't block ship.
6. **No automated map E2E.** Entire epic relies on PO manual QA on `myoko-kogen` (+ hakuba for ski_resort). Regression risk for layout/CSP/pin rendering grows with each follow-up story.
7. **Operational checklist items still open.** GCP billing budget alert (11.2 AC 20), `NEXT_PUBLIC_MAPBOX_TOKEN` on Vercel prod, curated content backfill for hakuba/myoko `ski_resort` entries — PO/Ops tasks, not code blockers.
8. **Google Places cost assumptions updated mid-epic.** Shard corrected March 2025 billing change; 11.8 radius reduction further reduces API call surface — cost math in shard may need one-line refresh.

## Key Insights

1. **Google Maps URLs require both `query` and `query_place_id`.** Per [Google Maps URLs spec](https://developers.google.com/maps/documentation/urls/get-started#search-action), `query` is the fallback when place ID can't resolve — omitting it sends users to wrong locations.
2. **Don't mix Tailwind layout utilities with custom `grid-template-columns` on the same element.** Pick one owner; use `@media` scoping for breakpoint-specific column animation (11.9 lesson).
3. **Split acceptance criteria by breakpoint** when mobile and desktop behavior diverges — single "responsive layout" AC invites side-effect regressions.
4. **Mapbox GL JS requires `worker-src 'self' blob:` in CSP.** Build success does not prove map works — browser console check is mandatory for map stories.
5. **`"All"` is a selection state, not a category.** Don't leak fake enum values into places API or analytics — model as `ChipSelection = 'all' | MapChipKey` (11.7).
6. **Attraction/Transportation Google categories are ski-resort noise.** Curated `ski_resort` entries cover valley sub-areas better than `tourist_attraction` Nearby Search.
7. **Auto-fit bounds matters more than search radius for perceived pin density.** 11.8 PO diagnosis: pins felt "too far" because viewport didn't reframe, not because 5 km was too wide.
8. **Amend-not-reopen for superseded decisions.** When 11.8 changed radius from 11.2's 5 km, leave 11.2 record as historical truth; new story owns current values.
