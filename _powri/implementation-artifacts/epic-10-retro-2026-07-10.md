## What Went Well

1. **`navConfig.ts` as navigation single source of truth.** Split into `MOBILE_BOTTOM_NAV_ITEMS`, `DESKTOP_CENTRE_NAV_ITEMS`, `DESKTOP_HAMBURGER_ITEMS`, `MOBILE_HAMBURGER_ITEMS` in 10.2 — prevented drift between bottom nav, desktop bar, and hamburger menus across six stories.
2. **Epic 9 auth patterns integrated without friction.** `useAuth()`, `openSignIn({ trigger: 'nav' | 'saved' })`, session cookies, and avatar display in desktop nav worked on first pass in 10.1; 10.3/10.4 extended `SavedResortsProvider` inside existing `AuthProvider` without architectural rework.
3. **PO-driven UX polish elevated desktop IA.** Story 10.2's unified single-bar desktop nav (hamburger + Powri at viewport edges, centred browse links) matched full-bleed hero layout — a meaningful product improvement beyond the epic shard's original two-row interim design.
4. **Deep root-cause documentation on 10.2.1.** Tailwind v4 static-scan limitation on `${CONST}` interpolation in arbitrary-value brackets is now documented in `pageInsets.ts` header + source-text regression guard in `pageInsets.test.ts` — prevents recurrence across future layout work.
5. **E2E reliability lesson institutionalized (10.2.2).** `reuseExistingServer: false` + dedicated port 3012 + testing-strategy guardrail against `git stash` baseline checks on stale servers — fixes a class of false "pre-existing failure" claims that masked real vs infra issues.
6. **Saved-resorts dual-storage model shipped complete.** Guest localStorage, authenticated `/api/saved`, optimistic UI, shared-device merge rules (0-row gate), logout hygiene, and cross-device sync — full FR-15.1/FR-15.2 scope in 10.3 + 10.4.
7. **`TopToast` extraction.** Shared toast component from `AuthSuccessToast` pattern enabled consistent save/unsave/merge feedback with correct durations (5s save with `/saved` navigation, 3s unsave/merge).

## Challenges & Growth Areas

1. **PO visual sign-off on dev mode masked production CSS bug.** Story 10.2 PO approved 375px title alignment while `pageInsets.ts` interpolated classes compiled to zero CSS rules in production builds — titles appeared aligned relative to each other but all sat under the fixed bar. Lesson: layout stories require `npm run build` + CSS artifact verification, not dev-mode-only QA.
2. **Follow-up story cascade from desktop-focused polish.** 10.2's viewport-edge nav fix (AC 18–21) shipped desktop-first; mobile title overlap (10.2.1) discovered immediately after. Future layout stories should include mobile + production-build checks in the same story's DoD before PO sign-off.
3. **E2E false positives eroded trust in test results.** 10.2.1's Dev Agent Record mislabeled 4 failures as "pre-existing product regressions" when they were stale `npm run dev` on port 3000. Required dedicated story 10.2.2 to correct. Baseline comparisons must use isolated servers.
4. **Scope grew via PO additions mid-epic.** 10.2 gained desktop IA redesign + Info→About + UX polish ACs; 10.3 gained heart chrome/toast specs; 10.4 gained shared-device merge rules. Valuable product decisions, but story files grew large — consider splitting PO polish into explicit follow-up stories earlier.
5. **Deferred edge cases documented but not resolved.** 10.4 code review deferred: `saved_count` analytics vs filtered list, merge TOCTOU on concurrent tabs, orphaned localStorage on cold session restore. Low severity but tracked in `deferred-work.md`.
6. **Signed-in E2E coverage thinner than guest.** 10.3/10.4 E2E focus on guest flows; authenticated merge + cross-device sync relies on unit/API tests. Manual or E2E auth specs still open for confidence before launch gate.

## Key Insights

1. **Never interpolate TS constants into Tailwind arbitrary-value brackets.** Use CSS custom properties (`--top-bar-h`) referenced by fully static class strings (`pt-[calc(var(--top-bar-h)+env(safe-area-inset-top))]`). Tailwind v4's scanner reads raw source text, not evaluated JS.
2. **Fixed bar height tokens must include `border-b`.** `--top-bar-h` at `3.5rem` matched `h-14` content but not the 1px `border-divider` — caused deterministic 1px `AppMenuSheet` gap. Document "rendered footprint" not "content box" in token comments.
3. **Playwright `reuseExistingServer` is unsafe for correctness locally.** Ambient `npm run dev` on the default port causes cold-route compilation stalls (33s+) that surface as `ERR_ABORTED` — not reproducible in production or isolated `next start`.
4. **`navConfig` composition scales.** Adding a nav destination means updating one config file + i18n + analytics destination enum — not hunting four components.
5. **Shared-device merge needs explicit PO rules.** "Merge only when account has 0 DB rows" prevents guest localStorage from overwriting an existing account's saves on library/hotel computers — product decision, not obvious from FR text alone.
6. **Provider optimistic state enables toast→list freshness.** Prepending slug on save before toast shows means `/saved` navigation needs no artificial delay or refetch — pattern worth reusing for other list features (reviews, trips).
