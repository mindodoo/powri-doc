## What Went Well

1. **CS → DS → CR workflow scaled cleanly.** Nine stories shipped sequentially with no rework cascades; patch stories (2.2.1, 2.7.1) absorbed UX feedback without destabilising prior work.
2. **Foundation paid off immediately.** Design tokens, i18n, content loader, and analytics from Epic 1 meant every Epic 2 screen used consistent patterns from day one.
3. **Client-side discovery stack is cohesive.** Filters, search, and quiz all operate on the same resort dataset with instant UI updates — no full page reloads.
4. **Quiz scoring is testable and documented.** `features/quiz/scoring.md` + persona fixtures (Ninja, Alex, Mind, etc.) gave confidence through 2.7 and 2.7.1 without manual regression every time.
5. **Content gate cleared.** All 20 resorts published — Home and Explore show real data, not empty states.
6. **Spec drift handled with patch stories.** When quiz results spec moved to podium-only, Story 2.7.1 closed the gap without reopening 2.7.

---

## Challenges & Growth Areas

1. **Spec updated after initial 2.7 ship.** Quiz results originally used full `ResortList`; planning docs updated to podium top 3 mid-epic. Lesson: lock results UX in `features/quiz/brief.md` before DS on complex flows.
2. **Resort card images still placeholders.** Gradient fallbacks acceptable for dev, but Home/Explore/quiz podium lack photography until Unsplash fetch pipeline runs (Epic 3 dependency).
3. **Explore search not wired.** Story 2.5 deferred Explore top-bar search to a polish pass; EXPERIENCE.md lists search on both tabs.
4. **Resort detail still stub.** Card taps land on placeholder detail pages — expected (Epic 3), but demos need framing until 3.1 ships.
5. **Accessibility deferrals accumulated.** Onboarding keyboard nav, `aria-labelledby`/`id` mismatch, and some tap-target audits deferred across 2.1–2.4 — bundle into Epic 3 polish or dedicated a11y pass before launch gate.
6. **Analytics context inconsistencies.** Quiz podium uses 1-based `position`; Home list uses 0-based — intentional but document in `tracking-plan.md` to avoid dashboard confusion.
7. **Nested git still unresolved.** Same carry-forward from Epic 1; blocks proper PR workflow and deploy hygiene.

---

## Key Insights

1. **Patch stories are a valid BMAD pattern.** 2.2.1 (hero height) and 2.7.1 (podium) absorbed post-ship UX without reopening closed stories — keep using `X-Y-1` naming for spec deltas.
2. **Feature briefs prevent quiz rework.** `features/quiz/brief.md` as single source for Serious/Cosmic copy, reveal timing, and podium layout made 2.7.1 scope crisp.
3. **Deferred CR findings are technical debt inventory.** ~25 defer items across Epic 2 stories form a ready backlog for Epic 3 prep and launch polish — no need to re-discover them.
4. **Published content unlocks real UX validation.** Epic 1 retro warned about empty Home; Epic 2 with 20 published resorts enables meaningful filter/search/quiz testing.
5. **Shared-element transition is wired early.** Story 2.3 named transition targets; Epic 3 detail hero (3.1) can plug in without reworking card taps.
