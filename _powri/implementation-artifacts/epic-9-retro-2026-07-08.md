## What Went Well

1. **Solid architectural foundation.** Stories 9.1‚Äì9.2 established migrations, RLS, `@supabase/ssr` cookie sessions, and `verify:supabase` before any UI ‚Äî downstream stories built cleanly on that base.
2. **Bug stories captured institutional knowledge.** Each prod issue (9.6‚Äì9.14) includes symptom ‚Üí code trace ‚Üí fix ‚Üí files involved. Future agents can debug auth without re-discovering PKCE, Site URL, or storage pitfalls.
3. **PKCE lesson applied systemically (9.8).** Magic link initiation moved to browser client, matching Google OAuth. The architectural mismatch (server `signInWithOtp` vs cookie-based callback) is now documented as a pattern to avoid.
4. **Prod/staging separation (9.7) prevented recurrence.** Splitting `powri-dev` and `powri-prod` with per-project auth URLs fixed the P0 localhost redirect and isolates QA data from production users.
5. **Auth UX matured through iteration.** Toast on sign-in (9.11), Google account picker (9.10), OAuth error page (9.12), desktop modal polish (9.14) ‚Äî each small fix improved first-impression conversion surface (SM-6).
6. **`verify:supabase` as ops safety net.** Extended in 9.13 to probe avatar upload/read ‚Äî catches prod misconfiguration that table-existence checks miss.
7. **GDPR flows shipped complete.** Export JSON shape, deletion request + ban, re-login block, and analytics event ‚Äî ready for Epic 15.3 legal copy updates.

## Challenges & Growth Areas

1. **Production testing lagged behind "done" stories.** Story 9.3 manual E2E was deferred at dev-complete; P0 bug 9.6 appeared only after prod deploy. Auth epics need prod smoke as a story gate, not a retro afterthought.
2. **Single-project assumption (9.1) was wrong for launch.** Reusing one Supabase project across local/preview/prod caused auth URL conflicts. 9.7 superseded 9.1's PO decision ‚Äî planning should default to env-separated projects for auth-bearing epics.
3. **Supabase Dashboard config is part of the product.** Site URL vs Redirect URLs vs `/**` wildcards (9.6, 9.7, 9.12) caused three separate bugs. Ops checklist in `deploy-environment.md` was updated reactively; should be a mandatory pre-prod gate from day one.
4. **Magic link production delivery blocked (9.9).** Built-in Supabase email caps at 2/hour/project. Google OAuth is the reliable prod path until custom SMTP + owned domain. Magic link works architecturally (9.8) but cannot be QA'd at volume on prod.
5. **Bug cascade expanded scope 180%.** Planned 5 stories became 14. Valuable hardening, but Epic 10+ scheduling should assume auth-adjacent epics may spawn prod bug stories.
6. **Staging/preview manual E2E still open.** Multiple stories defer full Google + magic link E2E on Preview URLs. Sprint marks stories done; checklist items remain unchecked in artifacts.
7. **Story artifact vs sprint-status drift.** 9.8 and 9.10 story files still show `Status: review` with open manual prod ACs while sprint-status shows `done`. Artifacts should be synced at story close.

## Key Insights

1. **Auth initiation and callback must share cookie storage.** Any Supabase PKCE flow (OAuth, magic link) must use `@supabase/ssr` browser client for initiation ‚Äî server-side `signInWithOtp` cannot work with cookie-based callback.
2. **Supabase Site URL is not a redirect wildcard.** `https://domain/**` as Site URL produces literal `/**` paths on OAuth errors (404). Site URL = origin only; Redirect URLs = exact callback paths.
3. **Prod/staging Supabase split is non-negotiable for launch.** Auth URL config, user data, and email quotas must be isolated per environment.
4. **`verify:supabase` should grow with storage features.** Bucket existence ‚âÝ upload permissions. Service-role upload probes catch prod 500s before users do.
5. **Post-sign-in UX is part of auth completion.** `?auth=signed_in` firing analytics is insufficient ‚Äî users need visible confirmation and new-user onboarding routing.
6. **Responsive auth UI needs breakpoint-specific layout rules.** Shared mobile bottom-sheet classes (`max-h-[90dvh] overflow-y-auto`) break desktop modal UX (9.14).
