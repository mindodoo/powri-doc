## What Went Well

1. **PO-driven scope stayed tight.** Stories 6.1–6.6 were pure UX polish with no schema explosions; each shipped in one CS → DS → CR cycle.
2. **Content + code split cleanly.** 6.7/6.7b/6.8/6.9 were mostly editorial with small supporting scripts (Unsplash fetch, link gate tweaks).
3. **Unsplash pipeline is repeatable.** `npm run fetch:unsplash` + CDN map + attribution on list cards means future season refreshes don't need hand-editing 20 markdown files.
4. **Launch gate evolved without breaking CI.** Optional `trail_map_url`, bot-block documentation, and link probe refinements kept `test:launch:links` useful without false-blocking deploy.
5. **Epic 6 closed Epic 5 debt.** Images, daytrips, link verification, PostHog registry, and deploy docs were all flagged in Epic 5 retro — all addressed here.
6. **Late story 6.12 landed fast.** One token + line-clamp + content rename — good template for post-PO-review micro-fixes without reopening Epic 3.

---

## Challenges & Growth Areas

1. **Official resort URLs are unreliable for automation.** Joetsu, Kandatsu, and Lotte block probes and often browsers — omitting trail map sections was the right product call vs. dead links.
2. **Unsplash fetch had edge cases.** Hero array comment syntax bug and card→hero repair needed a follow-up commit (6.7b) — script needs `--repair-hero-from-card` documented in runbook.
3. **PostHog dashboards are partly manual.** Wizard created core funnels; filter guardrail, link CTR breakdown, and satisfaction avg still require one-time pin in UI (recipes in §7).
4. **`last_reviewed` still empty.** Editorial sign-off dates remain the largest content-process gap before wide public share.
5. **Quick verdict was oversized by design.** UX-DR7 specified `display-italic` (24px) on a fixed 55vh hero — long copy + attribution stacked until 6.12; future content guidelines should cap quick verdict length.
6. **6.6 icon scope churn.** Per-item bullet icons were reverted to header-only per PO — good reminder to confirm cosmetic scope before batch-editing 20 resorts.

---

## Key Insights

1. **Pre-launch epic is the right place for PO feedback.** Epic 6 absorbed discovery flow changes that would have been awkward mid–Epic 2 without destabilising core architecture.
2. **Trail map is optional when official sites fail.** Hiding the section beats linking to 403 pages — launch gate should warn on broken URLs, not require a URL per resort.
3. **Image pipeline belongs in repo tooling.** Editorial Unsplash work at scale needs a script + env key at repo root, not manual frontmatter edits.
4. **Hero overlay needs typographic hierarchy.** Title > verdict > attribution — verdict must be subordinate in size and clamped in lines on mobile.
5. **Retrospective action items from Epic 5 drove Epic 6 backlog.** Mapping RETRO-* IDs to stories kept prioritisation traceable.
